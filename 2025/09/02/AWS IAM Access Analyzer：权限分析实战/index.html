AWS IAM Access Analyzer：权限分析实战
559 字 2 分钟

在云上运维中，安全的核心是 权限控制。
即便我们给资源加了 IAM 策略或安全组，也难以确认 “是不是意外对外开放了？”。

AWS 提供的 IAM Access Analyzer 就是用来解决这个问题的 —— 它能自动分析账号中的 IAM 策略，告诉你资源是否被外部实体访问。

1️⃣ 什么是 IAM Access Analyzer？

  • IAM Access Analyzer 是 AWS 提供的 自动化权限分析工具
  • 它会扫描你的 IAM 策略，检查是否允许外部访问（比如：
    • S3 存储桶对所有人可读？
    • KMS 密钥是否被外部账号使用？
    • IAM 角色是否能被第三方 Assume？

📌 一句话：Access Analyzer = 帮你找出“资源是否被外部共享”。

2️⃣ 支持的资源类型

目前 IAM Access Analyzer 能分析的资源包括：

  • S3 存储桶
  • IAM 角色
  • KMS 密钥
  • Lambda 函数 & Layer
  • SQS 队列
  • Secrets Manager

3️⃣ 如何使用 IAM Access Analyzer

第一步：进入控制台

  • 打开 IAM 控制台 → 左侧选择 Access Analyzer

第二步：创建 Analyzer

  • 点击 Create analyzer
  • 选择类型：
    • Account（分析整个账号的外部访问）
    • Organization（如果你用 Organizations，可以分析整个组织）
  • 填写名称，比如：MySecurityAnalyzer

第三步：查看分析结果

  • 创建后，Analyzer 会自动开始扫描
  • 如果检测到有外部访问，会在 Findings 列表里显示
  • 每条 Finding 都有详情，比如：
    • 资源类型：S3 Bucket
    • 外部实体：arn:aws:iam::123456789012:role/ExampleRole
    • 允许的操作：s3:GetObject

4️⃣ 实际应用场景

  • S3 存储桶是否公开
  • IAM 角色是否能被外部 Assume
  • KMS 密钥是否被共享给外部账号
  • 合规审计：证明资源权限受控

5️⃣ 最佳实践

✅ 在每个账号里创建至少一个 Analyzer（建议 全局区域 us-east-1）
✅ 定期检查 Findings，确认是否是预期行为
✅ 对非预期的 Findings：

  • 修改策略 → 限制访问
  • 标记为 Resolved（已修复）
    ✅ 配合 CloudWatch Events，在检测到新 Findings 时自动触发告警

🔚 总结

  • IAM Access Analyzer 是 AWS 提供的 权限体检工具
  • 它能帮你自动发现“意外对外开放”的资源
  • 创建 Analyzer 免费，检查结果也免费，非常适合日常安全运维
