SSH 连接服务器全攻略
612 字 2 分钟

SSH（Secure Shell）是管理 Linux 服务器的标准方式，既能用于 AWS EC2，也能用于 普通物理机 / VPS。本文将分两部分详细介绍。

一、AWS EC2 实例 SSH 连接

1. 前提条件

  • 已启动一台 EC2 Linux 实例。
  • 创建时下载了 密钥对（.pem 文件）。
  • 本地安装了 SSH 客户端（Mac/Linux 默认有，Windows 可用 PowerShell 或 PuTTY）。

2. 获取实例公网 IP

  • 进入 EC2 控制台 → Instances
  • 查看 Public IPv4 address（如 54.210.xx.xx）。

3. 调整密钥权限（Mac/Linux）

4. 连接命令

  • -i my-key.pem → 指定私钥文件
  • ec2-user → 默认用户名（Amazon Linux）。
    • Ubuntu → ubuntu
    • CentOS → centos

成功后进入命令行界面：

5. 注意事项

  • 安全组需放行 **TCP 22 (SSH)**，来源设置为 My IP。
  • 用户名要和系统对应，否则报错 Permission denied (publickey)。

二、普通 Linux 服务器 SSH 连接

如果你使用的是 IDC 物理服务器 或 云厂商（阿里云、腾讯云、华为云等） 的 VPS，连接方式稍有不同：

1. 获取服务器信息

  • 公网 IP（例如 123.45.67.89）。
  • 用户名/密码 或 私钥文件。

2. 密码方式登录

最常见的情况：厂商直接提供用户名和密码。

  • root 是默认管理员账户。
  • 系统会提示输入密码。

3. 密钥方式登录

如果厂商提供了 私钥文件（.pem 或 .key）：

4. Windows 用户（PuTTY）

  • 直接输入 IP + 用户名，用密码登录；
  • 如果是密钥，则先用 PuTTYgen 转换成 .ppk 再登录。

三、常见问题排查

错误信息 可能原因 解决办法
Permission denied (publickey) 用户名错误 / 私钥不匹配 确认系统默认用户，检查密钥对
Connection timed out 防火墙或安全组未放行端口 22 在云控制台放行 TCP 22
UNPROTECTED PRIVATE KEY FILE! 私钥文件权限过宽 chmod 400 file.pem
登录后中文乱码 服务器缺少本地化设置 安装 locales 包，设置 UTF-8

四、总结

  • AWS EC2 → 通常用 密钥对（.pem 文件） 登录，用户名是 ec2-user/ubuntu/...。
  • 普通服务器/VPS → 多用 root + 密码，也支持密钥认证。
  • 安全要点：
    • SSH 端口（22）只允许自己的 IP 访问。
    • 建议配置 MFA / SSH Key，避免弱口令被暴力破解。
