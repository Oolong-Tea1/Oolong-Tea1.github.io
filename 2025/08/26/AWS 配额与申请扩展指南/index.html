AWS 配额与申请扩展指南
833 字 2 分钟

在日常使用 AWS 的过程中，你可能会遇到这样的问题：

  • 想要启动更多的 EC2 实例，却提示“配额不足”；
  • 想创建多个弹性 IP（EIP），结果发现最多只能有 5 个；
  • 部署多个环境时，VPC 数量受限。

这些情况其实都是 AWS 配额（Service Quotas） 的限制。本文将带你全面理解配额，并手把手教你如何申请扩展。

一、什么是 AWS 配额（Service Quotas）

AWS 配额指的是 AWS 账号在某个服务下能使用的最大资源数量或速率。

  • 软限制（Soft Limit）：默认值较低，可以申请提升（例如 EC2 实例数量、EIP 数量）。
  • 硬限制（Hard Limit）：无法提升，受 AWS 架构约束（例如单个安全组最多规则数）。

常见默认配额示例：

  • EC2：单个区域内默认 20 个实例
  • EIP：每个区域最多 5 个
  • VPC：每个区域默认 5 个
  • RDS：默认最多 40 个数据库实例

👉 不同服务、不同区域的配额各不相同。

二、为什么要关注配额

  1. 避免上线受阻：如果在业务高峰时才发现资源不够用，会严重影响进度。
  2. 合理规划架构：提前了解配额，能避免因限制导致的部署失败。
  3. 企业合规需求：大规模应用通常会涉及数十个 VPC、上百个实例，需要提前扩容。

三、如何查看当前配额

AWS 提供了 Service Quotas 控制台，你可以在里面查看和管理所有服务的配额。

操作步骤：

  1. 登录 AWS 控制台，搜索并进入 Service Quotas。
  2. 选择需要查看的服务，例如 EC2。
  3. 在列表中即可看到该服务的配额详情（名称、默认值、可否调整）。

四、如何申请提升配额

如果你发现某个配额不足，可以通过控制台发起申请。

建议是在全英文界面下进行申请。

操作步骤：

  1. 打开 Service Quotas → 进入目标服务（例如 EC2）。
  2. 找到要扩展的配额（如 Running On-Demand Standard instances）。
  3. 点击 Request quota increase。
  4. 输入目标值（如从 20 提升到 50）。
  5. 提交申请后，AWS 会自动生成支持工单，等待审批。

五、常见的需要提升的配额场景

  • EC2 实例数：企业级应用需要更多计算节点
  • EIP 数量：多环境、多服务的公网访问需求
  • VPC 数量：多租户架构或区域隔离部署
  • CloudFormation 堆栈资源限制：大规模自动化部署时常遇到

六、最佳实践

  1. 提前申请
    在业务上线前至少一周提交扩容申请，避免紧急情况。
  2. 注意区域性
    配额是按 Region 区分的，比如东京区域的配额扩展不会影响新加坡区域。
  3. 监控与自动化
    使用 Service Quotas API 或 CloudWatch 定期检查配额，避免超限。
  4. 企业支持计划
    建议启用 Business/Enterprise Support，审批速度更快，更适合大规模团队。

七、总结

  • 配额是 AWS 的资源使用上限，既是保护机制，也是规划依据。
  • 查看 → 申请 → 扩展 是标准流程，通常 1–2 天内即可完成。
  • 建议所有团队在正式部署前 预估配额需求并提前申请，避免生产环境卡壳。
