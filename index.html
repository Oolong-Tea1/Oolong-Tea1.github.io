AWS 中 IAM MFA 设置教程：从零开始管理用户与权限~
636 字 2 分钟


在 AWS 中，IAM（身份与访问管理） 是最基础也最重要的安全配置。
通过 IAM，你可以创建独立的用户、分配合适的权限，避免所有人都用 root 账号操作 AWS，从而降低安全风险。

下面，我们通过几个步骤来完成 IAM 基础配置。

1️⃣ 创建 IAM 用户

进入 IAM 控制台，选择 Add user。
输入用户名（例如 user），选择 AWS Management Console access，并设置初始密码。
这样，新建的用户就可以通过独立账号登录 AWS，而不用使用 root 账号。

2️⃣ 创建用户组（Group）并分配权限

最佳实践是通过 用户组来管理权限。
例如，创建一个 user-group 组，然后为它附加策略 AmazonEC2FullAccess。
这样，所有加入这个组的用户都能管理 EC2 实例，而不用逐个分配权限。

3️⃣ 将用户加入用户组

接下来，把 user 加入 user-group 组。
用户组的策略会立即生效，这样用户就继承了组的所有权限。

5️⃣ 开启 MFA（手机扫码登录）

为了提高账号安全性，建议给 IAM 用户启用 MFA（多因素认证）。
具体操作步骤：

  1. 打开用户的 Security credentials 页面，点击 Assign MFA device
  2. 选择 Authenticator app（推荐），系统会生成一个 二维码
  3. 打开手机上的 Google Authenticator、Microsoft Authenticator、1Password 等 TOTP 应用，扫码绑定
  4. 应用会生成一个动态的 6 位验证码（30 秒刷新一次）
  5. 在 AWS 控制台输入两次连续的验证码，验证成功后绑定完成

👉 之后，每次用该 IAM 用户登录 AWS 控制台时，除了输入密码，还需要输入手机上的验证码，相当于 手机扫码 + 动态口令登录，大大提高了账号安全性。

6️⃣ 最佳实践小结

  • 不要用 root 账号日常操作
  • 每个人都有自己的 IAM 用户
  • 使用用户组来批量管理权限
  • 给敏感操作配置 最小权限原则
  • 开启 MFA，保证账号安全
